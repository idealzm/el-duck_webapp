# Настройка Админ-Панели VPN Сервиса

## Быстрый старт

### 1. Настройка Telegram бота

1. Создайте бота в [@BotFather](https://t.me/BotFather)
2. Получите токен бота
3. Откройте `bot_config.json` и укажите токен:

```json
{
    "botToken": "1234567890:ABCdefGHIjklMNOpqrsTUVwxyz",
    "botUsername": "your_bot"
}
```

### 2. Настройка администраторов

Откройте `admin_config.json` и добавьте ваш Telegram ID:

```json
{
    "adminIds": ["123456789"],
    ...
}
```

**Как узнать свой Telegram ID:**
- Напишите боту [@userinfobot](https://t.me/userinfobot)
- Или используйте API: `https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getUpdates`

### 3. Настройка домена для авторизации

Для работы Telegram Login Widget нужно указать домен в настройках бота:

1. Откройте [@BotFather](https://t.me/BotFather)
2. Отправьте `/mybots`
3. Выберите вашего бота
4. Нажмите `Bot Settings` → `Telegram Login Widget`
5. Укажите домен вашего сайта (например: `dev.el-duck.ru`)

## Доступ к админ-панели

Откройте в браузере: `https://ваш-домен.com/admin.html`

## Функционал

### Пользователи
- Просмотр всех пользователей
- Поиск по ID, username, имени
- Изменение баланса (+/−)
- Управление подпиской
- Удаление пользователя

### Цены
- Цена Telegram Proxy — **влияет на основной сайт**
- Цена Полного доступа (VPN + Proxy) — **влияет на основной сайт**
- Лимиты пополнения (мин/макс)

### Подписки
- Статистика по подпискам
- Список всех активных подписок
- Статус каждой подписки

### Настройки
- Включение/выключение сайта
- Сообщение при тех. работах
- URL конфигурации WireGuard — **влияет на инструкции в приложении**
- URL установщика Windows — **влияет на инструкции в приложении**
- Proxy сервер — **влияет на ссылку подключения в приложении**
- Proxy порт — **влияет на ссылку подключения в приложении**
- Proxy пользователь — **влияет на ссылку подключения в приложении**
- Proxy пароль — **влияет на ссылку подключения в приложении**
- Список администраторов

## Безопасность

- Авторизация только через Telegram Login Widget
- Проверка хеша авторизации на сервере
- Сессионные токены (24 часа)
- Проверка прав администратора для каждого запроса

## Файлы

| Файл | Описание |
|------|----------|
| `admin.html` | Страница админ-панели |
| `admin.css` | Стили админ-панели |

## Примечания

- Все данные хранятся локально на сервере
- Сессии действуют 24 часа
- Для продакшена рекомендуется настроить HTTPS

## Решение проблем

### Ошибка "Invalid auth data"

1. Проверьте, что домен в браузере совпадает с доменом в настройках бота
2. Убедитесь, что токен бота правильный в `bot_config.json`
3. Проверьте логи PHP для деталей ошибки

### Ошибка 401

Проверьте PHP логи:
```bash
tail -20 /var/log/nginx/error.log
# или
tail -20 /var/log/php_error.log
```
